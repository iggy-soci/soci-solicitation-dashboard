⚙ Admin profile

1 Add a solicitation survey

When a new solicitation survey is deployed in Pendo, add it here. Fill the fields and press Add survey — everything else is automatic: the survey is committed to the repo, its product card appears on the dashboard, and the data refreshes within a couple of minutes.

?If checked, the token is kept in this tab's session storage and cleared when you close the tab. Leave unchecked on a shared computer.

2 Live surveys

Surveys currently pulling data from Pendo. Removing one takes its card and queue entries off the dashboard on the next refresh — nothing is deleted in Pendo, and re-adding the same guide ID restores it. Uses the same GitHub token as section 1. ?Remove dispatches the repo workflow with your token, which edits surveys.json and triggers a refresh — the mirror of adding. If the token field in section 1 is empty and not remembered, paste it there first.

3 Change admin password

Type a new password to get its hash, then paste the line over the ADMIN_PW_SHA256 constant near the top of this file's script and re-deploy. ?The password is stored only as a SHA-256 hash in the page, never in plain text. Changing it requires editing the file and re-deploying, since a hosted static page can't save settings by itself.


    


    

      
4 How data refreshes

      
This dashboard reads dashboard_data.json, which a GitHub Action rebuilds from Pendo on a schedule. User levels (Administrator / Manager / User) come straight from Pendo, so there is nothing to refresh manually. To change how often it updates, edit the cron in .github/workflows/refresh.yml.
